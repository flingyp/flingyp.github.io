聊聊关于MVVM设计思想

发表于 2020-02-18 | 分类于 Vuejs | 本文总阅读量 次

一. 传统的设计思想

大家都可以回想下自己初学web前端的时候就单纯的用 HTML CSS JS为了写一个页面是有多繁琐。在写界面时，我们经常会碰到一些功能。需要通过数据来生成页面。

这个时候整个流程就是 通过页面来收集数据 ，有了数据在生成相应的页面。当数据改变后再次刷新页面 再将新的数据渲染到页面上。

实际上整个过程就做了两件事

  • 通过页面的一些操作 获取数据和变更数据
  • 根据数据生成或者改变界面

但是就为了做这两件事就够我们去折腾的了就更边提业务方面的功能实现了。

二.MVVM横空出世

MVVM的出现让我们的开发方便了许多，相信大家使用过 Vue 都可以体验到。没错MVVM提供了一套配置帮助我们将数据和界面用线连着谁也离不开谁。

三.MVVM构成

MVVM的全名是Model-View-ViewModel

  • Model 是 模型的意思
  • View 是 观察 视图的意思
  • ViewModel 就是观察模型 视图模型的意思了

很好理解 model 其实就是 指的是 数据 view 就是视图了展示给用户的页面

而MVVM就是将两种结合联系起来 形成的ViewModel

就拿Vue举例来说：

Vue作为前端三大框架之一 深受大家的喜爱但是这是为什么呢？ 它和MVVM思想有什么必要的联系呢？

Vue.js 官方对自己的介绍是这样说的：

Vue (读音 /vjuː/，类似于 view) 是一套用于构建用户界面的渐进式框架。与其它大型框架不同的是，Vue 被设计为可以自底向上逐层应用。Vue 的核心库只关注视图层，不仅易于上手，还便于与第三方库或既有项目整合。另一方面，当与现代化的工具链以及各种支持类库结合使用时，Vue 也完全能够为复杂的单页应用提供驱动。

这里有一句很重要的话： 只关注图层 易于上手。

这就是为什么大家都会去学Vue这个框架的原因之一了。

四.Vue 和 MVVM的联系

总结下来： Vue的特点之一就是MVVM思想， 数据驱动视图 非常灵活

当我们修改存放的数据那么视图也就会随之改变 同理 视图的改变如果影响到数据那么数据也就会改变。 两者就完全联系在一起。并且 Vue的数据驱动视图的思想解放了开发者对DOM的操作。 也就是说在学习了 MVVM 和 Vue 之后 大家对于JS 或者 Jquery库 的使用频率会大大减少 ，整个页面开发不需要DOM的操作就可以实现对页面的更新 并且是响应式的，让开发者完全投入业务的开发。
